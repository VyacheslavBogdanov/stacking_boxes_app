# Техническое задание: Stacking Boxes App

## Общие требования

-   Пользовательский интерфейс приложения должен быть на русском языке.
-   Одно окно для ввода исходных данных и вывода результатов расчёта.
-   При запуске приложения все поля должны быть пустыми для исключения получения данных предыдущего расчёта.

## Вводные данные

### Поля ручного ввода (4 поля)

1. **Длина**, мм
2. **Ширина**, мм
3. **Высота**, мм
4. **Вес брутто**, кг

### Выпадающий список (1 поле)

**Марка картона** — выпадающий список, значения которого загружаются с сервера.

Начальные значения:

-   Т11
-   Т23
-   Т24

#### Управление списком марок картона

-   Список марок картона хранится в JSON-файле на сервере.
-   **Клиент (без авторизации)** — видит выпадающий список и может только выбирать из него для расчёта.
-   **Администратор (с авторизацией)** — входит через форму логина, после чего получает доступ к управлению списком: добавление, редактирование, удаление марок картона.
-   Для авторизации администратора используется форма логин/пароль.
-   Изменения списка доступны всем пользователям сразу после сохранения.

## Результаты расчёта

Три поля (строки) с результатами:

1. **Максимальная высота штабеля**, мм
2. **Количество рядов коробок**, шт
3. **Максимальный вес на коробку**, кг
